## 二、爆款指数公式

> 两个平台算法不同，爆款指数公式也不同。

### 小红书爆款指数

```
爆款指数 = 赞藏率 × 0.5 + 评论率 × 0.3 + 涨粉率 × 0.2

其中：
  赞藏率 = (点赞数 + 收藏数) / 阅读量
  评论率 = 评论数 / 阅读量
  涨粉率 = 涨粉数 / 阅读量
```

**解释**：小红书是"种草-收藏"逻辑，所以赞藏权重最高（0.5）。评论代表深度互动（0.3）。涨粉代表转化价值（0.2）。

### 抖音爆款指数

```
爆款指数 = 完播率 × 0.4 + 互动率 × 0.4 + 涨粉率 × 0.2

其中：
  完播率 = 完整播放次数 / 总播放量
  互动率 = (点赞数 + 评论数 + 分享数 + 收藏数) / 播放量
  涨粉率 = 涨粉数 / 播放量
```

**解释**：抖音是"内容消费-推荐流"逻辑，完播率决定是否继续推流（0.4），互动率决定推荐强度（0.4），涨粉是附加价值（0.2）。

### 判定阈值（初始假设，待数据校准）

| 等级 | 小红书爆款指数 | 抖音爆款指数 | 意义 |
|------|---------------|-------------|------|
| 爆款 | > 0.15 | > 0.20 | 明显超出账号平均水平，重点拆解 |
| 小爆 | > 0.08 | > 0.12 | 表现良好，可作为内容方向参考 |
| 常规 | 0.03 - 0.08 | 0.05 - 0.12 | 正常水平，维持 |
| 低迷 | < 0.03 | < 0.05 | 内容/选题/发布策略需复盘 |

> 注意：以上阈值基于赛道通用经验假设，需积累 20+ 条作品数据后重新校准。

## 三、数据更新流程

### 每条作品发布后

```
发布后1小时 → 记录初始数据（播放量、点赞、评论）
发布后3小时 → 第二次记录，观察增速
发布后24小时 → 核心数据节点，填写完整数据行
发布后72小时 → 最终数据收尾（如果还有明显增长则延长观察）
```

### 每周复盘

```
每周日：
  1. 导出本周数据，计算每条爆款指数
  2. 标注本周最优和最差作品
  3. 对比不同内容类型、发布时间、钩子类型的平均表现
  4. 输出结论：什么类型的下周多做 1 条，什么类型暂停
  5. 更新对标账号本周爆款（如有）
```

### 每月复盘

```
每月1日：
  1. 汇总月度数据，输出粉丝增长趋势图
  2. 重新校准爆款指数阈值（如果数据量够）
  3. 评估三大内容支柱比例是否偏离策略
  4. 月报摘要写入 .claude/memory/status/
```

## 四、快速检查清单（发布后必答）

每次作品发布后24h，用一个3分钟的快速复盘：

- [ ] 这条的前3秒钩子是什么类型？数据反馈好吗？
- [ ] 评论区TOP3评论是什么主题？有选题灵感吗？
- [ ] 发布时间的数据表现和之前比如何？
- [ ] 这条如果数据好，成功因素最可能是哪个？如果差，最可能的原因是什么？
- [ ] 下一条要不要调整什么？
