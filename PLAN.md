# 猫猫动画 IP 项目 — 整体规划

> 目标：数据驱动的 AI 猫猫内容工厂 → 小红书+抖音分发 → IP 变现
>
> 参与人：你（架构师/AI 研发）+ 朋友（运营中枢/业务专家）
> 核心理念：数据说话、真相优先（Truth First）、小步快跑

---

## 〇、前置探索清单（Phase 0 之前必须做）

> 这些是决定项目能否跑通的关键问题。不探清楚就开工 = 白干。

### 必探项

| # | 探索题 | 怎么做 | 决策标准 |
|---|---|---|---|
| Q1 | **猫猫角色一致性** | 在小云雀上测试同一个 prompt 生成 10 次，看形象是否稳定 | 花纹/色块/脸型偏差 < 20% 即通过 |
| Q2 | **小云雀内容合规边界** | 生成一版猫猫视频试发小红书，看是否被限流/下架 | 24h 内正常曝光即通过 |
| Q3 | **小红书+抖音个人号发布规则** | 手动发 3 条不同来源的视频（AI/混剪/实拍），观察流量差异 | 明确 AI 内容是否有流量歧视 |
| Q4 | **数据获取可行性** | 测试禅妈妈/新榜/灰豚免费版能拿到什么数据 | 拿到基础榜单数据即满足 |
| Q5 | **官方飞书 CLI 入库可用性** | 完成 `lark-cli` 授权，并向飞书多维表格或电子表格写入一条测试记录 | 能结构化写入即通过 |

### 探完再决定的事

- 要不要上自动化发布？（如果 Q3 发现 AI 视频被限流，那自动化没意义）
- 要不要买第三方数据工具？（如果 Q4 免费版够用就不买）
- Base Prompt 要不要锁定固定 Seed？（Q1 结果决定）

## 三、Agent 体系设计（9 个 Agent）

| # | Agent | 职责 | 关键约束 |
|---|---|---|---|
| ① | **趋势/竞品 Agent** | 读飞书多维表格中的爆款数据，分析趋势 | ⚠️ **不做实时抓取**，只读结构化数据 |
| ② | **选品决策 Agent** | 趋势 + 历史爆款指数 → 今日选题 | 输出选题卡（含"为什么选这个"） |
| ③ | **文案 Agent** | 为选题出两套文案（小红书风 / 抖音风） | 两套完全不同的语气、格式、标签 |
| ④ | **生产 Agent** | 调小云雀生成 + 自动消重/二创 | 必须加滤镜/贴纸/转场扰动 |
| 🛑 | **Gatekeeper** | 飞书群推审核卡片 | 未得人工确认绝不往下走 |
| ⑤ | **发布 Agent** | 前期推卡片让人手动发，后期自动 | P1-P2 手动，P4 自动化 |
| ⑥ | **数据回收 Agent** | 每天录入各平台互动数据 | 前期人工填表，后期 API |
| ⑦ | **复盘 Agent** | 按爆款指数公式分析，出策略建议 | 不只看播放量 |
| ⑧ | **飞书 Bot** | 群交互、转指令、推卡片、推报告 | 所有 agent 的统一入口 |
| ⑨ | **规划 Agent** | Claude Code，你用来写代码调架构 | 你掌控 |
| ⑩ | **账号策略 Agent** | 定期读复盘数据 → 更新策略文档 | 对应 `skills/01-账号策略维护/SKILL.md` |
| ⑪ | **封面包装 Agent** | 封面大字/视觉/标题/标签/多平台适配 | 对应 `skills/04-封面包装/SKILL.md` |

### 策略文档层（单事实源）

所有 Agent 和 Skill 的底层数据源：
- `content/账号策略.md` — 阿饱IP定位、人设、内容方向、禁区
- `content/粉丝画像.md` — 目标粉丝特征、需求、平台差异
- `content/爆款拆解经验.md` — 同赛道爆款规律拆解

- `skills/02-选题推荐/SKILL.md` → Agent ②
- `skills/03-脚本创作/SKILL.md` → Agent ③
- `skills/04-封面包装/SKILL.md` → Agent ⑪
- `skills/05-发布复盘/SKILL.md` → Agent ⑦

### 工具链
- `tools/codex-automation/` — Codex 自动化数据采集+日报生成
- 官方 `lark-cli`（`@larksuite/cli`）— 飞书多维表格/电子表格结构化入库（Q5 验证项）

## 四、关键风险与对策

### 风险 1：数据采集被反爬

| 阶段 | 方案 |
|---|---|
| **P0-P2** | 放弃全自动爬虫。人工在手机上刷 + 第三方平台（禅妈妈/新榜/灰豚）免费榜单 → 录入飞书多维表格 |
| **P3+** | 评估是否购买第三方 API（约 200-500元/月）。不做网页端实时抓取 |

**飞书多维表格作为唯一数据中枢：**
```
朋友在手机上刷到爆款 → 复制链接 → 填入多维表格
                               │
                 ┌─────────────┼─────────────┐
                 ▼             ▼             ▼
            Agent ① 读     Agent ⑥ 写    Agent ⑦ 算
            做趋势分析     录每日数据    出复盘报告
```

### 风险 2：AI 视频被平台限流/判搬运

| 环节 | 对策 |
|---|---|
| **生产** | Agent ④ 自动加随机扰动：滤镜强度 ±10%、贴纸位置随机偏移、转场类型随机选 |
| **发布** | Phase 1-3 一律手机端手动发布，摸清平台风控边界后再自动化 |
| **测试** | 前置探索 Q3：发 3 条不同来源视频对比流量，明确 AI 内容的流量天花板 |

### 风险 3：Agent 幻觉导致低质内容公开

| 环节 | 对策 |
|---|---|
| **Gatekeeper** | Agent ④ 出视频后 → 飞书群推审核卡片 → 必须人工点 [允许] 才往后走 |
| **降级** | 如果朋友不在线，视频进入待审核队列，绝不过期自动发 |

### Phase 3：完整运营飞轮 + 复盘体系

| # | Agent | 上线标准 |
|---|---|---|
| P3.1 | ① 趋势/竞品 Agent | 每日自动分析多维表格数据，出趋势报告 |
| P3.2 | ② 选品 Agent | 结合趋势+爆款指数出选题卡 |
| P3.3 | ③ 文案 Agent | 每个选题出两套完整发布包 |
| P3.4 | ④ 生产 Agent（半自动） | 调小云雀 + 消重/二创 |
| P3.5 | ⑤ 发布 Agent（手动+审核） | 飞书推卡片 → 朋友手机发 |
| P3.6 | ⑥+⑦ 数据+复盘 Agent | 每日日报 + 爆款指数计算 + 策略建议 |

**复盘指标体系：**

| 平台 | 核心指标 | 爆款指数公式 |
|---|---|---|
| **小红书** | 点赞率、收藏率、评论率 | 赞藏率×0.5 + 评论率×0.3 + 涨粉率×0.2 |
| **抖音** | 3秒停留率、完播率、互动率 | 完播率×0.4 + 互动率×0.4 + 涨粉率×0.2 |

**P3 完成标志：** 飞轮自动转。你每天只需要在飞书群看报告 + 点审核卡片。
