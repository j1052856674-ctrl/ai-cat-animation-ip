# 封面包装

## 一、解决什么问题

短视频平台上，封面和标题是用户在信息流中的第一接触点——"点击率 = 封面吸引力 × 标题好奇心 × 算法分发"。即使内容质量极高，封面和标题不行，播放量直接折损 70%+。本 Skill 基于脚本大纲中的核心信息，设计高点击率的封面文案、跨平台标题、描述和配图提示词。

## 二、需要读取什么

- `content/账号策略.md`：账号视觉风格、人设调性、色彩倾向
- `content/粉丝画像.md`：粉丝年龄/性别/审美偏好 → 决定视觉风格（年轻化/质感/极简/冲击）
- `content/爆款拆解经验.md`：已验证的封面大字公式、标题结构、高 CTR 模式
- `03-脚本创作` 产出的脚本大纲：开头钩子、核心信息、情绪基调 → 封面和标题的创作素材

## 三、用户需要提供什么输入

- 触发词："做封面"、"出封面"、"封面和标题"
- 必须输入：对应作品的脚本大纲（或从"上一轮脚本创作"直接衔接）
- 可选约束：希望的色调/风格、需要避开的关键词、品牌元素要求

## 四、输出什么

### 4.1 封面大字方案（3 个方向）
每个方案包含：
- **核心大字**（6-10 字）：提炼脚本最大卖点，用爆款封面公式（数字冲击/痛点反问/结果承诺/矛盾冲突）
- **副文案**（可选）：辅助信息（如"第 3 点太真实"、"看完沉默了"）
- **排版建议**：文字位置、大小比例、与图片主体的关系

### 4.2 跨平台标题
- **小红书标题**（≤20 字）：干货感 + 利他性 + emoji 点缀
- **抖音标题**（≤30 字）：情绪强 + 钩子猛 + 话题标签建议
- **B站标题**（≤40 字，可选）：信息量更大、含关键词利于搜索

### 4.3 跨平台描述
- **小红书描述**：正文文案 + 末尾标签（#话题1 #话题2 ...）
- **抖音描述**：简介文案 + 话题标签 + @引导
- **B站简介**（可选）：更完整的说明文案

### 4.4 封面图 AI 提示词
一段可直接输入 Midjourney / Stable Diffusion / DALL-E 的 AI 生成提示词，包含：
- 主体描述
- 风格（参考账号视觉风格）
- 构图（留白位置留给封面大字）
- 色彩倾向
- 情绪/氛围（匹配脚本情绪基调）

## 五、什么时候使用

- 脚本完成后："做封面"、"出封面"、"封面和标题"
- 标准工作流：`脚本创作 → 封面包装 → 创作者发布`
- 也可以独立使用：已有成品视频需要重做封面时，提供视频的核心信息即可

## 六、和其他 Skill 的关系

```
03-脚本创作（产出: 脚本大纲 → 含核心信息 + 情绪基调）
    ↓
04-封面包装 ← content/账号策略.md, content/粉丝画像.md, content/爆款拆解经验.md
    ↓
创作者发布
```

- **上游**：`03-脚本创作` —— 脚本大纲是封面和标题的核心素材来源
- **下游**：创作者直接使用封面文案、标题、描述进行发布
- **职责边界**：只出封面方案和文案，不实际制作封面图（那是设计工具的活）。AI 提示词是给设计工具用的，本 Skill 不生成图片

## 七、注意事项

1. **封面大字和标题不能重复**：大字是视觉冲击（短、猛、一眼抓住），标题是信息补充（长一点、有好奇点）。两者说同一句话浪费了一半点击率
2. **平台差异必须体现**：小红书的标题偏干货和清单感，抖音的标题偏情绪和冲突感。不能用同一套标题通发
3. **AI 提示词必须留白**：生成的图片要为封面大字留出摆放空间（如"主体偏右/偏下，左侧/上方留白"），否则文字放上去遮挡主体
4. **描述文案的最后一句是转化指令**：小红书描述末尾引导收藏/评论，抖音描述末尾引导点赞/关注。不要把转化指令放在中间被折叠
5. **色彩一致性**：封面主色调应与账号已有的视觉风格一致，避免每条作品封面色调完全不同，破坏主页美观度和品牌识别度
6. **标签策略**：小红书 5-10 个标签（精准长尾 + 大词各半），抖音 2-3 个标签（挑最核心的）。不是标签越多越好
