# 选题推荐

## 一、解决什么问题

内容创作者常有"不知道下一条拍什么"的困境。本 Skill 基于账号策略、粉丝数据、爆款规律和历史表现，一次生成 5-10 条选题建议，让创作者的每条作品都有明确的选题依据而非凭直觉，提高爆款概率，避免选题同质化。

## 二、需要读取什么

- `content/账号策略.md`：账号定位 + 内容方向 + 差异化卖点
- `content/粉丝画像.md`：粉丝特征 + 兴趣标签 + 活跃时段（决定发布时间窗口）
- `content/爆款拆解经验.md`：已验证有效的爆款公式、标题结构、钩子模式、封面套路
- `content/历史作品数据库.md`：已发作品记录（标题、类型、平台、各维度数据），用于避免重复 + 判断哪些方向已跑通

## 三、用户需要提供什么输入

- 触发词："推荐选题"、"下一条拍什么"、"给我几个选题"
- 可选约束：指定平台（小红书/抖音/B站/全平台）、指定内容类型（口播/vlog/教程/剧情）、指定情感调性（温暖/搞笑/深度/犀利）、指定时长（短/中/长）

## 四、输出什么

**一份选题卡列表（5-10 条）**，每条选题卡包含：

| 字段 | 说明 |
|------|------|
| 标题 | 候选标题（模仿爆款标题结构） |
| 适合平台 | 小红书 / 抖音 / B站 / 全平台 |
| 内容类型 | 口播 / vlog / 教程 / 剧情 / 混剪 |
| 为什么适合 | 基于策略/画像/爆款规律的论证 |
| 封面方向 | 封面核心文字 + 视觉方向建议 |
| 优先级 | 高 / 中 / 低（按：匹配策略度 + 爆款概率 + 制作成本综合判定） |
| 风险提示 | 可能翻车的原因（如争议性强、时效窗口短） |

输出形式：结构化表格，可读性强，创作者可直接从中挑选。

## 五、什么时候使用

- 内容枯竭时："不知道拍什么"
- 新周期开始时：做完复盘，策略更新后需要新方向
- 同类型作品数据连续下滑，需要尝试新方向
- 用户说"推荐选题"、"下一条拍什么"、"给我几个选题方向"

标准工作流：`02-选题推荐 → 创作者选定 → 03-脚本创作`

## 六、和其他 Skill 的关系

```
01-账号策略维护（产出: 策略文档）
    ↓
02-选题推荐 ← content/粉丝画像.md, content/爆款拆解经验.md, content/历史作品数据库.md
    ↓
03-脚本创作（接收: 创作者选定的选题卡）
```

- **上游**：`01-账号策略维护` —— 选题方向受策略约束；`content/` 下的数据文件提供事实基础
- **下游**：`03-脚本创作` —— 创作者选定某条选题卡后，传递给脚本创作
- **职责边界**：只推荐选题，不写脚本。不替创作者决定选哪个，只做优先级排序和理由说明

## 七、注意事项

1. **避免重复**：必须核对 `content/历史作品数据库.md`，确保推荐选题不与近期作品高度雷同
2. **尊重策略约束**：如果策略文档明确说了"不做某类内容"，选题绝对不能推荐该类
3. **时效性选题谨慎推荐**：热点型选题标注时效窗口，提醒创作者抓紧或放弃
4. **优先级不是绝对命令**：创作者有最终决策权，本 Skill 只是提供数据驱动的排序建议
5. **新品种草期策略**：新账号前 5 条作品，推荐覆盖 2-3 个不同方向做测试，而非锁定单一方向
