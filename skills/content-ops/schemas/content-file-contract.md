# Content File Contract

> 定义 `content/` 下每个策略/数据文件的最小结构契约。各 Skill 解析这些文件时的假设。
> 本文件不复制内容，只声明契约。权威内容在 `content/*.md` 中。

---

## 通用 frontmatter 契约

所有 `content/*.md` 必须包含以下 frontmatter 字段：

| 字段 | 类型 | 必须 | 说明 |
|------|------|:----:|------|
| `name` | string | ✅ | 机器可读标识（英文kebab-case） |
| `description` | string | ✅ | 文件用途一句话 |
| `version` | string | ✅ | 语义版本号 x.y.z |
| `confidence` | string | ✅ | 数据可信度：假设版 / 调研模板 / 部分验证 / 已验证 |
| `created` | date | ✅ | 创建日期 YYYY-MM-DD |
| `status` | string | ✅ | 当前状态 |
| `metadata.type` | string | ✅ | strategy / research / reference |
| `metadata.bridge` | string | ✅ | "true" — 标记可被 memory-bridge 拉取 |

## 各文件专属契约

### 1. `content/账号策略.md`

**章节必需**：
- 一句话定位
- 核心人设（含名字/物种/视觉特征/性格/口头禅/声音的维度表）
- 内容主线（含三大内容支柱+占比表）
- 适合/不适合的内容类型表
- 平台差异策略表（至少含小红书和抖音）

**能力方依赖**：01-strategy、02-topic、03-script、04-cover 均读取。

### 2. `content/粉丝画像.md`

**章节必需**：
- 核心人群画像（年龄/性别/地域/职业/兴趣）
- 活跃时段
- 关注动机
- 版本与更新时间

**能力方依赖**：02-topic、03-script、04-cover、05-review 均读取。

### 3. `content/爆款拆解经验.md`

**章节必需**：
- 已验证的爆款公式列表
- 标题结构模式
- 钩子模式（反问式/反差式/数据冲击式/痛点共鸣式等）
- 封面套路
- 版本与更新时间

**能力方依赖**：02-topic、03-script、04-cover 均读取。

### 4. `content/历史作品数据库.md`

**章节必需**：
- 数据字段规范表（基础信息+内容属性+平台指标+评论分析）
- 爆款指数公式（小红书版+抖音版）
- 判定阈值表
- 数据更新流程
- 快速检查清单
- 状态标注（当前数据量阶段）

**能力方依赖**：02-topic 读（避免重复）、05-review 读（趋势对比）。

### 5. `content/竞品调研数据库.md`

**章节必需**：
- 调研目标（至少4个问题）
- 候选IP方向对比表
- 关键词池（小红书+抖音分平台）
- 采样规则（最小样本量+筛选条件）
- 数据字段表
- 评分公式（内容热度分+归一化热度+IP方向匹配分）
- 决策阈值（进入试发+固化方向）
- 调研输出格式

**能力方依赖**：00-competitor-data 读（字段定义+采样规则）。

## 解析假设（各 Skill 必须遵守）

| # | 假设 | 影响方 |
|---|------|--------|
| 1 | frontmatter 是 YAML，用 `---` 包裹 | 所有 Skill |
| 2 | 表格用 Markdown 管道符格式 `| col | col |` | 所有 Skill |
| 3 | 平台差异至少存在小红书和抖音两列 | 02/03/04/05 |
| 4 | 版本记录在文档末尾，格式 `| 版本 | 日期 | 变更 |` | 01-strategy |
| 5 | 数值字段使用纯数字，不含千分位逗号 | 05-review |
| 6 | 百分比以小数或 `%` 标记均可；计算前需归一化 | 05-review |
| 7 | 日期统一 `YYYY-MM-DD` 格式 | 所有 Skill |
| 8 | `content/` 文件始终是权威源；Skill 输出的策略更新必须写回对应文件 | 01-strategy |
