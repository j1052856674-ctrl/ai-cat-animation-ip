# Module 04: 封面包装

> 将 03-脚本创作 产出的脚本大纲转化为可直接用于发布的封面方案、跨平台标题、描述文案与 AI 配图提示词。

## Purpose

短视频信息流中，封面和标题决定点击率（CTR = 封面吸引力 × 标题好奇心 × 算法分发），直接决定播放量的 70%+。本模块连接"内容准备好了"到"可以发布了"之间的关键一步：把内容卖点翻译成视觉和文案信号，让目标用户在 0.5 秒内产生点击欲望。

## Inputs

| 输入 | 来源 | 必须 | 说明 |
|------|------|------|------|
| `script_outline` | 03-脚本创作 输出 | 是 | 脚本大纲：开头钩子 + 分段大纲 + 情绪基调 + 核心信息 |
| `platform_target` | 用户指定 | 否 | 目标平台（默认全平台：小红书 + 抖音）；单一平台则跳过多平台标题/描述 |
| `tone_constraint` | 用户指定 | 否 | 希望的色调/风格约束（如"更犀利"、"更治愈"、"要冲击感"） |
| `forbidden_keywords` | 用户指定 | 否 | 需要避开的关键词或表达方式 |
| `brand_elements` | 用户指定 | 否 | 必须包含的品牌/IP元素要求（如阿饱必须露脸、必须暖色调） |

## Reads

按以下优先级读取，后一项覆盖前一项的缺省：

1. **`content/账号策略.md` -- 标题与封面方向 章节**：账号视觉风格（暖黄+白+深灰）、封面原则（阿饱脸部占50%+）、标题公式（6种可套用）、配色规范、IP辨识度要求
2. **`content/粉丝画像.md`**：粉丝年龄/性别/审美偏好 → 决定视觉风格方向（年轻化/质感/极简/冲击）和语言风格（年轻人用语/职场用语）
3. **`content/爆款拆解经验.md`**：已验证的封面大字公式（数字冲击/痛点反问/结果承诺/矛盾冲突）、标题结构、高CTR模式、标签策略

## Outputs

输出一份 **cover-package**（发布包-封面分册），包含以下 4 个子模块：

### 4.1 封面大字方案（3 个方向）

每个方案含：

| 字段 | 说明 |
|------|------|
| `main_text` | 核心大字（6-10 字），提炼内容最大卖点，套用爆款封面公式 |
| `direction_type` | 所用公式：数字冲击 / 痛点反问 / 结果承诺 / 矛盾冲突 |
| `sub_text` | 副文案（可选），辅助信息（如"第3点太真实"、"看完沉默了"） |
| `layout_hint` | 排版建议：文字位置、大小比例、与图片主体的空间关系 |

### 4.2 跨平台标题

| 平台 | 长度限制 | 风格要求 | 字段名 |
|------|----------|----------|--------|
| 小红书 | ≤20 字 | 干货感 + 利他性 + emoji 点缀 | `title_xiaohongshu` |
| 抖音 | ≤30 字 | 情绪强 + 钩子猛 + 口语化 | `title_douyin` |
| B站 | ≤40 字 | 信息量大、含搜索关键词 | `title_bilibili`（可选） |

**规则**：封面大字和标题不可重复——大字做视觉冲击，标题做信息补充。两者说同一句话浪费一半点击率。

### 4.3 跨平台描述

| 平台 | 内容要求 | 字段名 |
|------|----------|--------|
| 小红书 | 正文文案 + 末尾标签（#话题1 #话题2 ...）+ 转化指令放末尾 | `desc_xiaohongshu` |
| 抖音 | 简介文案 + 话题标签（2-3个）+ @引导（如有联动） | `desc_douyin` |
| B站 | 更完整说明文案 | `desc_bilibili`（可选） |

**关键规则**：描述文案最后一句必须是转化指令（小红书：收藏/评论引导；抖音：点赞/关注引导），不要放在中间被折叠。

### 4.4 封面图 AI 提示词

输出一段结构化提示词对象，可直接输入 Midjourney / Stable Diffusion / DALL-E：

| 字段 | 说明 |
|------|------|
| `subject` | 主体描述（如"阿饱橘猫，豆豆眼、黑眼圈、小领带"） |
| `style` | 风格约束，参考账号视觉风格（如"手绘感、粗糙感、暖黄色调、不精美"） |
| `composition` | 构图指引，明确留白位置给封面大字（如"主体偏右/偏下，左侧/上方留白 40%"） |
| `color` | 色彩倾向（暖黄+白+深灰为主） |
| `mood` | 情绪/氛围，匹配脚本情绪基调（丧→疲倦氛围 / 治愈→温暖光线） |

### Output Schema

```yaml
cover_package:
  package_id: "cover-<YYYYMMDD>-<序号>"
  script_id: ""
  cover_texts:           # 3 directions
    - main_text: ""

      direction_type: ""
      layout_hint: ""

    douyin: ""
    bilibili: ""         # optional
  descriptions:
    xiaohongshu: ""
    douyin: ""
    bilibili: ""         # optional
  ai_prompt:
    subject: ""
    style: ""
    composition: ""
    color: ""
    mood: ""
  tags:
    xiaohongshu: []      # 3-5, 精准长尾 + 大词各半

```

## Procedure

```
Step 1: 读取 content/账号策略.md 的标题与封面方向章节 → 确定视觉风格基线（配色、字体、IP辨识度要求）
Step 2: 读取 content/粉丝画像.md → 确定审美偏好方向（年轻化/冲击/极简）
Step 3: 读取 content/爆款拆解经验.md → 调取已验证的封面公式和高CTR模式
Step 4: 从 script_outline 中提取：① 核心卖点（可转化为大字的一句话）② 情绪基调（丧/治愈/共鸣/愤怒）③ 开头钩子类型
Step 5: 生成 3 个封面大字方案 → 每个套用不同爆款公式，避免同质化
Step 6: 生成跨平台标题 → 小红书/抖音各一版，B站可选。确保与大字不重复
Step 7: 生成跨平台描述 → 描述末尾放转化指令，不放在中间
Step 8: 生成 AI 提示词 → 确保留白、风格匹配账号策略、情绪匹配脚本基调
Step 9: 输出 cover-package → 标注推荐方向及理由
```

## Missing Input Handling

| 缺失项 | 处理方式 | 降级后能否继续 |
|--------|----------|---------------|
| `script_outline` 完全缺失 | **不直接卡死**。引导用户：① 先走 03-脚本创作 产出大纲；② 或手动提供核心信息（内容主题 + 情绪基调 + 核心卖点一句话），基于精简输入降级生成封面方案。降级模式下标注 `coverage: degraded (missing_full_script)` | 是（降级模式） |
| `content/账号策略.md` 不存在 | 降级为通用封面设计原则（不绑定 IP 视觉风格），标注 `missing_brand_guidelines`。建议用户优先创建账号策略文档 | 是（降级） |
| `content/粉丝画像.md` 不存在 | 降级为通用粉丝画像（默认 22-35 岁打工人），标注 `missing_persona_data` | 是（降级） |
| `content/爆款拆解经验.md` 不存在 | 降级为通用封面公式（数字/反问/对比），标注 `missing_proven_formulas`。封面方案在"有数据验证的公式"维度可信度降低 | 是（降级） |
| 用户指定单一平台 | 只输出该平台的标题和描述，跳过其他平台。`next_step` 中标注 | 是（范围缩减） |

## Safety / Boundaries

1. **封面大字和标题不能重复**：大字是视觉冲击（短、猛、一眼抓住），标题是信息补充（长一点、有好奇点）。两者说同一句话浪费一半点击率。自检时必须逐个对比大字和标题，确保不重复。
2. **平台差异必须体现**：小红书偏干货和清单感，抖音偏情绪和冲突感，不能用同一套标题通发。每个平台的标题和描述必须独立生成。
3. **AI 提示词必须留白**：生成的图片必须为封面大字留出摆放空间（如"主体偏右/偏下，左侧/上方留白 40%"），否则文字放上去遮挡主体。
4. **描述文案最后一句是转化指令**：小红书末尾引导收藏/评论，抖音末尾引导点赞/关注。转化指令不放在中间避免被信息流折叠。
5. **色彩一致性**：封面主色调应与账号已有视觉风格一致，避免每条封面色调完全不同，破坏主页美观度和品牌识别度。参考 `content/账号策略.md` 配色规范。
6. **标签策略分层**：小红书 5-10 个标签（精准长尾 + 大词各半），抖音 2-3 个标签（最核心）。不是标签越多越好。
7. **不出实际封面图**：AI 提示词是给设计工具用的，本模块只出文字方案，不生成图片文件。不要承诺"生成封面图"。
8. **IP 辨识度优先**：任何封面方案必须确保 IP 主体（阿饱）脸部占画面 50% 以上、表情与标题情绪一致。不得为了"好看"牺牲 IP 辨识度。
9. **人设禁忌不可越界**：不得在封面文字和标题中出现攻击性语言、绝望表达、涉政涉敏内容。必须符合阿饱"丧完治愈"的底线。
