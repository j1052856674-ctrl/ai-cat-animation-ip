# Module 01 — 账号策略维护

## Purpose

内容 IP 账号运营一段时间后，粉丝画像可能偏移、平台算法偏好变化、内容赛道的竞争格局改变。本模块定期检视当前策略是否仍然有效，在复盘数据支持下做出"不变 / 微调 / 大调"的判定，输出更新后的策略文档。

**关键约束：新人期不启用深度策略调整，必须依赖复盘证据。** 没有经过 `05` 复盘的结论，不能修改策略。

## Inputs

| 字段 | 必填 | 说明 |
|------|:--:|------|
| `trigger` | 是 | 触发信号：`"update_strategy"` / `"periodic_review"` / `"post_review"` |
| `review_source` | 条件必填 | 复盘结论来源；`trigger=post_review` 时必须提供复盘 run_id 或最新复盘摘要 |
| `user_concerns` | 否 | 创作者补充的担忧或调整方向建议（仅供参考，不能作为唯一依据） |

## Reads

| 文件 | 用途 |
|------|------|
| `content/账号策略.md` | 当前策略基线：账号定位、目标人群、内容方向、发布节奏、平台策略 |
| `content/粉丝画像.md` | 最新粉丝数据：年龄、性别、地域、活跃时段、兴趣标签 |
| 最新复盘结论 | 由 `05-发布复盘` 输出，包含是否需要调整策略的判定依据与数据支撑 |

## Outputs

更新后的 `content/账号策略.md`，必须包含：

1. **变更摘要**：本次修改了哪些条目（如 "目标人群：从 Z 世代扩展到 18-35 岁女性"）
2. **变更理由**：每条修改对应的数据支撑或复盘结论（格式：`来源于[复盘编号]的[具体结论]`）
3. **未变部分确认**：哪些策略保持不变，为什么暂不改
4. **后续关注点**：需要下次复盘重点验证的假设
5. **版本标记**：文档底部追加 `--- 更新时间: YYYY-MM-DD, 触发: 第X次复盘`

## Procedure

### Step 1: 前置门禁 — 新人期判定

检查以下条件是否满足：

| 条件 | 说明 |
|------|------|
| 作品数 >= 10 条 | 账号有足够样本做统计推断 |
| Codex 数据管线已跑通 | 复盘所需数据字段可获取 |
| 有至少一次完整复盘 | `05-发布复盘` 至少输出过一次有效复盘结论 |

**任一条件不满足 → 新人期模式：**
- 不执行策略调整。
- 输出结果 `status: "blocked"`，`blocking_reason` 说明新人期规则。
- 使用初版策略文档作为当前策略基线。

### Step 2: 读取当前策略与复盘结论

1. 读取 `content/账号策略.md`，提取当前策略基线。
2. 读取 `content/粉丝画像.md`，对比粉丝画像是否发生显著变化。
3. 读取最新复盘结论，提取策略更新建议与支撑数据。

### Step 3: 判定级别

| 级别 | 判定标准 | 动作 |
|------|------|------|
| 不变 | 复盘数据无明显异常信号，粉丝画像无漂移 | 确认当前策略有效，记录"为什么暂不改" |
| 微调 | 单维度需要优化（如发布时段、标题风格） | 修改对应条目，每条标注复盘编号 |
| 大调 | 战略方向偏移（如主力平台切换、人设重新定位） | 完整修订策略文档，需创作者确认 |
| 待观察 | 数据不足或结论模糊 | 不调整，标注待观察项和下次复盘验证点 |

### Step 4: 保守原则检查

- 每条策略变更必须标注 "来源于哪次复盘的哪条结论"。
- 不能凭感觉改策略。
- 如果策略文档明确说了 "不做某类内容"，不能反向放开，除非复盘数据有强证据支撑且创作者明确确认。
- 数据不足或结论模糊时，标注"待观察"，不强行调整。

### Step 5: 写入策略文档

1. 在 `content/账号策略.md` 中更新对应条目。
2. 文档底部追加版本标记。
3. 不修改其他 Skill/Agent 自行维护的副本——策略文档是权威源。

## Missing Input Handling

| 缺失项 | 处理 |
|------|------|
| 无复盘结论 | **不凭感觉调整**；输出 `status: "blocked"`，`blocking_reason: "缺少复盘结论，无法判定策略有效性"` |
| 复盘结论无策略建议 | 当前周期标注 "不变" 或 "待观察"，不自行建议修改 |
| `content/账号策略.md` 不存在 | 输出 `status: "blocked"`，提示需先创建初版策略文档 |
| `content/粉丝画像.md` 不存在 | 标记粉丝数据缺失，但可基于复盘结论做有限调整；输出中标注 "粉丝画像缺失，调整结论置信度降低" |
| 新人期（作品数 < 10） | 不启用策略维护；直接返回新人期 block 状态 + 预计可启用条件 |
| 数据管线未跑通 | 同新人期处理；Codex 数据管线是复盘质量的前提 |

## Safety / Boundaries

1. **不能凭感觉改策略**：每条策略变更必须标注数据来源（哪次复盘、哪条结论）；没有证据的改动 = 不允许。
2. **新人期硬门禁**：作品数 < 10 条 或 数据管线未跑通时，不启用策略调整。此时使用初版策略文档。
3. **策略文档是权威源**：所有下游模块（选题、脚本、封面）读取的策略都来自本模块产出的 `content/账号策略.md`；不要在其他地方维护策略副本。
4. **保守原则**：数据不足或结论模糊时标注 "待观察"，不强行调整。
5. **创作者最终决策权**：大调级别的策略变更，输出建议后需创作者确认。
6. **不反向修改禁止项**：策略文档中明确禁止的内容方向，除非复盘数据有强证据支撑且创作者确认，否则不能放开。
7. **版本可追溯**：每次更新后必须追加版本标记，方便回溯策略演进。
