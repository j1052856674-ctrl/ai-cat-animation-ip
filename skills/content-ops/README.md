# content-ops — 内容运营 Skill 套件

## 这是什么

`content-ops` 是 ai-cat-animation-ip 项目的内容运营 Skill 套件，把从竞品调研到选题、脚本、封面、发布复盘的完整内容创作链路封装为统一入口。对外你只看到一个"内容运营总控"，对内各环节保持独立模块、可独立演进。

## 谁适合用

- **项目新人**：从 `modules/06-onboarding.md` 开始，套件会检查环境、初始化模板、产出第一条内容
- **日常运营**：对总控说"出选题""写脚本""做封面""复盘"即可，无需关心内部路由
- **维护者**：修改单一模块（如选题逻辑）不影响其他环节

**数据流**：`content/` 目录下的文件是所有模块的共享数据层：
- `content/账号策略.md` — IP 定位与方向
- `content/粉丝画像.md` — 目标受众
- `content/爆款拆解经验.md` — 爆款规律
- `content/历史作品数据库.md` — 已发布作品
- `content/竞品调研数据库.md` — 竞品样本

所有模块读取 `content/` 文件，产出写入统一路径（由 `schemas/` 定义格式）。

## 如何使用

### 第一次使用

说"我第一次用，帮我启动"或"帮我开始内容运营"。总控会自动：

1. 执行新人检查流程（模块 06-onboarding）
2. 检查 `content/` 文件是否完整
3. 检查工具环境（OpenCLI、飞书、Chrome）
4. 不可用时走降级路径，产出最小可行的第一条内容

### 日常运营

直接说需求，总控自动路由：

| 你想做的事 | 怎么说 |
|-----------|--------|
| 看看竞品在做什么 | "竞品调研""采集数据""看效果数据" |
| 不知道该拍什么 | "推荐选题""下一条拍什么""选题建议" |
| 开始写脚本 | "写脚本""出脚本""这条怎么拍" |
| 做封面 | "做封面""出封面""封面和标题" |
| 分析效果 | "复盘""分析数据""看看昨天那条" |
| 调整方向 | "更新策略""调整方向""重新定位" |
| 一条龙 | "从选题开始""来一条完整的" |

### 中断后继续

说"继续""接着上次"，总控从上次中断点恢复。

### 自己看模块文档

如果你需要深入了解某个环节的具体方法论（如竞品采集的阈值设定、选题卡的结构要求），直接读 `modules/` 下对应的 `.md` 文件即可。但日常使用不需要。

## 安全与限制

- **不自动发布**：套件产出的选题/脚本/封面方案是内容包，发布由你手动完成
- **不绕过授权**：需要你已授权飞书/OpenCLI 等工具
- **内容安全**：不生成违规内容，发现风险主动提示
- **新人期保护**：作品数 < 10 时不启用复盘和策略维护，避免小样本误判
