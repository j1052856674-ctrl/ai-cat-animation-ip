# 脚本创作

## 一、解决什么问题

短视频的核心竞争力在脚本结构：前 3 秒决定完播率，中间节奏决定留存率，结尾互动决定转化和涨粉。本 Skill 把选题卡转化为可拍摄的结构化脚本大纲——不写逐字稿（限制创作者发挥），而是设计关键的骨架节点，让创作者知道每一段拍什么、怎么讲、为什么这样安排。

## 二、需要读取什么

- `content/账号策略.md`：账号人设、口播风格（温暖/犀利/幽默/理性）、内容调性
- `content/粉丝画像.md`：粉丝年龄/性别/偏好 → 决定语言风格、节奏快慢、案例选择
- `content/爆款拆解经验.md`：已验证的钩子模式（反问式/反差式/数据冲击式/痛点共鸣式）、分段节奏模板、结尾互动公式
- `02-选题推荐` 产出的选题卡：创作者选定要创作的那条选题

## 三、用户需要提供什么输入

- 触发词："写脚本"、"出脚本"、"这条怎么拍"
- 必须输入：选定的选题卡（或直接从"上一轮选题推荐"中指定第几条）
- 可选约束：希望强调某个卖点、希望避开某种表达方式、指定是否需要展示某种画面

## 四、输出什么

**结构化脚本大纲（非逐字稿）**，包含以下模块：

### 4.1 开头 3 秒钩子（3 个备选）
每个备选标注：钩子类型（反问/反差/数据冲击/痛点共鸣/悬念/情绪先行）+ 一句话钩子文案。创作者可从中选择或组合。

### 4.2 分段大纲（3-5 段）
每段包含：
- **讲什么**：核心信息/观点（一句话概括）
- **画面放什么**：画面建议（口播直拍 / B-roll素材 / 动画 / 文字卡片 / 场景切换）
- **为什么这样安排**：基于爆款经验的节奏论证（如"第 2 段放案例增加可信度，防止观众划走"）

### 4.3 口播要点
- 语气建议（轻松/严肃/兴奋/娓娓道来）
- 节奏提示（"这一段放慢语速"、"这里停顿 1 秒制造悬念"）
- 需要重音强调的关键词

### 4.4 结尾互动引导（3 个备选）
- 引导类型（提问互动 / 投票 / 关注理由 / 下期预告 / 评论区召唤）
- 一句话引导文案

### 4.5 双版本适配
以上大纲同步输出两套：
- **小红书体**：紧凑、干货密度高、口播口语化、强调利他性
- **抖音体**：节奏更快、钩子更猛、情绪更强、强调爽感/共鸣

## 五、什么时候使用

- 创定选题后："写脚本"、"出脚本"、"这条怎么拍"
- 标准工作流：`选题推荐 → 创作者选定 → 脚本创作 → 封面包装`
- 注意：本 Skill 不负责"出选题"（那是 `02-选题推荐` 的职责）

## 六、和其他 Skill 的关系

```
02-选题推荐（产出: 选题卡列表 → 创作者选定某条）
    ↓
03-脚本创作 ← content/账号策略.md, content/粉丝画像.md, content/爆款拆解经验.md
    ↓
04-封面包装（接收: 脚本大纲 → 设计封面和标题）
```

- **上游**：`02-选题推荐` 的选题卡是本 Skill 的必需要输入；`content/` 下的策略和画像文件提供调性约束
- **下游**：`04-封面包装` —— 脚本大纲中的核心信息、情绪基调、开头钩子直接指导封面设计
- **职责边界**：只出脚本大纲，不写逐字稿（创作者需要自由发挥空间）；不负责选题推荐或封面设计

## 七、注意事项

1. **不是逐字稿**：本 Skill 输出结构化大纲，目的是让创作者理解"为什么这样拍"，而不是照着念稿。逐字稿会让内容失去真实感
2. **钩子是第一优先级**：前 3 秒的钩子质量直接决定完播率，必须提供多个备选方案让创作者测试
3. **每段必须有"为什么"**：让创作者理解叙事逻辑，而不是机械执行。这是短视频团队的稀缺能力
4. **双版本是硬要求**：小红书和抖音的内容消费习惯差异大，不能用同一版脚本通发。至少要在节奏、钩子强度、结尾引导上做区分
5. **口播风格参考账号策略**：策略文档中定义的人设和调性必须贯穿始终，不能这条温柔下条犀利（除非策略在测试期）
6. **时长意识**：大纲分段数量基于平台特性——小红书 60-90 秒（3 段）、抖音 30-60 秒（3-4 段）、B 站 3-8 分钟（5+ 段）
