# 账号策略维护

## 一、解决什么问题

内容 IP 账号运营一段时间后，粉丝画像可能偏移、平台算法偏好变化、内容赛道的竞争格局改变。本 Skill 负责定期检视当前策略是否仍然有效，并在复盘数据支持下，做出"不变 / 微调 / 大调"的判定，输出更新后的策略文档。

## 二、需要读取什么

- `content/账号策略.md`：当前策略基线（账号定位、目标人群、内容方向、发布节奏、平台策略）
- `content/粉丝画像.md`：最新粉丝数据（年龄、性别、地域、活跃时段、兴趣标签）
- 最新一次复盘结论（由 `05-发布复盘` 输出）

## 三、用户需要提供什么输入

- 确认"这次要做策略检视"或"复盘后更新策略"
- 可选：补充你认为需要调整的方向或担忧

## 四、输出什么

**一份更新后的 `content/账号策略.md`**，包含：

1. **变更摘要**：本次修改了哪些条目（如"目标人群：从 Z 世代扩展到 18-35 岁女性"）
2. **变更理由**：每条修改对应的数据支撑或复盘结论
3. **未变部分确认**：哪些策略保持不变，为什么暂不改
4. **后续关注点**：需要下次复盘重点验证的假设

## 五、什么时候使用

- 每月固定复盘后
- 连续 3 条作品数据持续下滑
- 用户说"更新策略"、"调整方向"、"重新定位"
- 粉丝画像发生显著变化（涨粉量大但互动率下降，或新增粉群体与原有画像不匹配）

## 六、和其他 Skill 的关系

```
05-发布复盘（输出: 复盘结论 + 策略更新建议）
    ↓
01-账号策略维护（执行: 读策略文档 + 复盘结论 → 输出新策略）
    ↓
02-选题推荐（读: 新策略 → 调整选题方向）
```

- **上游**：`05-发布复盘` —— 本 Skill 不主动判断要不要改策略，而是基于复盘输出做决策
- **下游**：`02-选题推荐`、`03-脚本创作`、`04-封面包装` —— 都会读取本 Skill 产出的策略文档
- **职责边界**：本 Skill 不负责看数据判好坏（那是复盘的职责），只负责把复盘结论转化为策略文档的具体修改

## 七、注意事项

1. **不能凭感觉改策略**：每条策略变更必须标注"来源于哪次复盘的哪条结论"
2. **新人期暂不启用**：新账号需要至少 10 条作品数据 + Codex 数据管线跑通后才能有效复盘，进而触发策略调整
3. **策略文档是权威源**：所有下游 Skill（选题、脚本、封面）读取的策略都来自本 Skill 产出的 `content/账号策略.md`，不要在其他地方维护策略副本
4. **保守原则**：数据不足或结论模糊时，标注"待观察"，不强行调整
5. **版本意识**：每次更新后在文档底部追加 `--- 更新时间: YYYY-MM-DD, 触发: 第X次复盘`，方便追溯
