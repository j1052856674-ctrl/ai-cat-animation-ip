# 决策 001：用 libtv（liblib CLI）替代小云雀

> **状态**：已确认
> **日期**：2026-06-30
> **影响范围**：视频生成引擎、技术栈、成本结构

---

## 背景

当前项目使用「小云雀」作为视频生成工具，但存在以下问题：
- ❌ Web UI，无法程序化调用
- ❌ 无 API/CLI，无法集成到 Agent 工作流
- ❌ 角色一致性差（靠 prompt + seed，形象随机）
- ❌ 功能单一（仅文生视频，无图生视频、首尾帧等）
- ❌ 成本不透明（免费额度，无法预估）

## 决策

**用 libtv（liblib CLI）替代小云雀作为视频生成引擎。**

## 理由

| 维度 | 小云雀 | libtv | 结论 |
|---|---|---|---|
| 访问方式 | Web UI | CLI + API | ✅ libtv 可自动化 |
| 图生视频 | ❌ | ✅ | ✅ libtv 解决角色一致性 |
| 工作流编排 | ❌ | ✅ 画布连线 | ✅ libtv 支持复杂工作流 |
| 脚本/分镜 | ❌ | ✅ | ✅ libtv 内置 |
| 成本 | 免费（额度有限） | ~$0.05/条 | ✅ 可控、透明 |
| 稳定性 | 免费额度可能调整 | 商业化产品 | ✅ 更稳定 |

## 影响

- **技术栈**：新增 libtv CLI，需 Node.js 环境
- **成本**：预计单条视频 $0.05（vs 小云雀免费但无法自动化）
- **流程**：从"人工操作 Web UI"变为"Agent 自动调用 CLI"
- **风险**：需验证生成效果是否可接受

## 替代方案

| 方案 | 评估 | 结论 |
|---|---|---|
| 继续用小云雀 | 无法自动化，长期不可持续 | ❌ 拒绝 |
| 用可灵 API | 效果好，但价格较高 | ⚠️ 备选 |
| 本地部署 WAN 2.1 | 免费，但需 GPU | ⚠️ 备选 |
| **libtv** | 性价比高，集成好 | ✅ **采纳** |

1. P0 验证：安装 libtv → 验证图生视频 → 评估成本
2. 如验证通过，正式替换小云雀
