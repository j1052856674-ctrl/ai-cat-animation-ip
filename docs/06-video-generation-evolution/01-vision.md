# 01-愿景与目标

> **为什么要做这个事？** 解决当前视频生产的核心痛点，构建可自动化、可扩展、可审计的视频生成体系。

---

## 当前痛点（Before）

### 小云雀的局限性

| 痛点 | 具体表现 | 影响 |
|---|---|---|
| **无法自动化** | Web UI，无 API/CLI | 每次需人工操作，无法集成到 Agent 工作流 |
| **角色一致性差** | 靠 prompt + seed，每次形象随机 | 猫 IP 形象不稳定，无法建立品牌认知 |
| **功能单一** | 仅文生视频 | 无法做图生视频、首尾帧、视频参考等 |
| **无工作流编排** | 单点工具 | 无法串联"脚本→图片→视频→合成"的完整链路 |
| **成本不透明** | 免费额度，无法预估 | 无法做成本预算和 ROI 分析 |

### 现有技术栈

```
数据采集：OpenCLI ✅
飞书读写：lark-cli ✅
视频生成：小云雀 ❌（无法自动化）
封面包装：无 ❌
```

---

## 目标状态（After）

### 新架构核心能力

| 能力 | 实现方式 | 收益 |
|---|---|---|
| **猫 IP 一致性** | 外部模型生成固定图片 → libtv 图生视频 | 形象 100% 一致 |
| **全流程自动化** | libtv CLI + 多模型 Selector | Agent 自动驱动，无需人工 |
| **多种视频模式** | text2video / singleImage2video / frames2video / audio2video | 玩法多样 |
| **质量可管控** | Reviewer Skill + Checkpoint 机制 | 生成质量可控 |
| **成本透明** | Cost Tracker → 每条视频精确到分 | 可预估、可审计 |
| **快速切换模型** | models.yaml + presets.yaml | 一行配置切换模型 |

---

## 关键原则

1. **模型与 Skill 解耦**：Skill 只定义"做什么"，不定义"用哪个模型做"
2. **配置驱动**：通过 YAML 配置管理模型，不改代码即可切换
3. **快速切换**：一行环境变量切换预设策略
4. **成本透明**：每个 Provider 自带价格标签
5. **效果可对比**：支持 A/B 测试，记录效果评分
