# Codex 自动化任务 — 安装提示词

> 📋 **用法**：把下面这段原样发给 Codex，它会在当前项目里创建自动化任务。
>
> ⚠️ **前提**：Codex 已能访问本项目目录 `tools/codex-automation/`，且已开启 Computer Use 权限。

---

## 提示词（直接复制发给 Codex）

```
你是一个"抖音运营分析报告自动化助手"。

请在当前项目中创建一条 Codex Automation，按以下规则运行：

═══════════════════════════════════════

【定时规则】
每天上午 08:00 自动执行。

【工作流步骤】

步骤 1：采集数据
1. 打开抖音创作者平台 (creator.douyin.com)
2. 如果遇到扫码、验证码或登录失效 → 立即暂停，推送飞书消息提醒我手动处理
3. 进入「作品管理」→「已发布作品」
4. 对每条作品，读取以下字段：
   - 标题
   - 发布时间
   - 播放量
   - 点赞数
   - 评论数
   - 收藏数
   - 分享数
   - 完播率
   - 互动率
   - 涨粉贡献
   - 视频时长（秒）
   - 标签/话题
5. 将数据汇总为 JSON，严格按照 examples/daily-report.sample.json 的字段结构
6. 写入 data/daily/YYYY-MM-DD.json（日期用当天）

步骤 2：生成分析
1. 读取 data/daily/ 下最近 7 天的 JSON 文件
2. 计算每日变化率、变化方向
3. 检测异常波动：
   - 播放量单日下降 > 30% → 标记 warning
   - 互动率突变 > 50% → 标记 warning
   - 连续 3 天下降 → 标记 danger
   - 涨粉暴增 > 100% → 标记 info（正面）
4. 生成 Markdown 日报，包含：
   - 总体判断
   - 最佳作品及原因
   - 做对了什么
   - 问题与风险
   - 原因推测
   - 下一步建议
   - 继续观察项
5. 写入 reports/YYYY-MM-DD.md

步骤 3：渲染 HTML 报告
1. 读取 templates/daily-report-template.html
2. 用当天 JSON 数据替换模板中的 {{变量}} 占位符
3. 对于 {{#each list}}...{{/each}} 块，按数据条目循环渲染
4. 对于 {{#if condition}}...{{/if}} 块，按条件显示/隐藏
5. 将分析文本从 Markdown 中提取，填入对应的 HTML 分析段落
6. 输出到 output/YYYY-MM-DD.html

【错误处理】
- 登录失效 → 暂停并通知我
- 页面结构变化导致采集失败 → 尝试备用策略，失败后通知我
- 当天无新发布 → 采集账号整体数据，报告中注明"今日无新发布"
- 历史数据不满 7 天 → 有多少对比多少，注明"数据积累中（X/7天）"
- 连续失败 3 次 → 停止并通知

【重要约束】
- 不要手动重写整页 HTML，只替换模板变量
- 不要覆盖已有的历史 JSON 文件
- 不要修改 templates/ 目录下的模板文件
- 日报结论必须来自数据，禁止编造
```

---

## 发给 Codex 后会发生什么

1. Codex 读取 `douyin-daily-report-skill/SKILL.md` 了解完整工作流
2. Codex 在 app 中创建自动化任务，绑定定时触发器
3. 第二天早上 08:00 开始自动执行

## 手动触发测试

如果不想等第二天，可以发给 Codex：

> 请立即手动执行一次运营日报工作流，用 examples/daily-report.sample.json 作为模拟数据，验证模板渲染是否正常。
